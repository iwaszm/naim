# 多模态 AU、表情与 Appraisal Proxy 识别反馈系统（Web 端）

## 项目实施与架构计划书（修订版）

## 1. 项目定位

本项目旨在基于开源 MediaPipe Web 模型能力，构建一个浏览器端的多模态情感交互系统。系统通过摄像头实时捕获用户面部表情、头部姿态，并在后续阶段扩展到手部与身体姿态；在此基础上，将 MediaPipe 输出的 landmarks 与 blendshapes 延伸为：

1. **AU proxy 识别**：将 MediaPipe blendshapes 与关键 landmarks 映射为 FACS Action Units 的近似强度。
2. **基础表情识别**：基于 AU proxy 与 blendshape 组合，识别 happiness、sadness、surprise、anger、fear、disgust、contempt 等基础表情倾向。
3. **Appraisal proxy 估计**：基于 AU、表情、姿态、时间变化与实验上下文，估计 novelty、pleasantness、goal obstruction、control/power 等认知评估维度的启发式 proxy。
4. **反馈交互**：通过可视化面板与虚拟智能体，对用户状态进行实时反馈，包括表情镜像、提示语、姿态/语音反馈等。

重要说明：本系统不将 appraisal 维度宣称为从面部表情直接“测量”的心理真值，而是将其定义为 **基于多模态信号与上下文规则的 proxy estimate**。这一表述更符合技术可实现性，也更适合后续学术验证。

## 2. 总体可行性判断

项目作为 **MediaPipe 能力延伸、AU/表情识别、Appraisal proxy 估计与反馈交互 Demo/研究平台** 是可行的。

需要控制的边界：

- AU 与表情识别可以作为第一阶段核心目标。
- Appraisal 识别应先以 proxy/rule-based estimate 形式实现，再通过实验数据和用户自评逐步校准。
- CSV/JSON 导出属于可选数据模块，不作为 MVP 的硬依赖。
- 3D 虚拟人反馈应在核心识别链路稳定后接入，避免一开始同时承担感知、认知映射、3D 资产、实时渲染四类风险。

## 3. 系统整体架构

系统采用前端优先架构。核心感知、AU 映射、表情识别、Appraisal proxy 估计和反馈逻辑都可以在浏览器端完成。后续如果需要实验数据集中管理、多人实验或模型微调，可以增加可选后端。

### 3.1 数据流

1. **输入层 Input**
   - Web Camera 视频流。
   - 初期仅启用 face stream；后续扩展 pose/hand。

2. **感知层 Perception**
   - 使用 `@mediapipe/tasks-vision`。
   - 第一阶段核心：`FaceLandmarker`。
   - 输出：478 个 face landmarks、face transformation matrix、52 个 blendshapes。
   - 后续扩展：`PoseLandmarker`、`HandLandmarker`。

3. **特征层 Feature Layer**
   - 对 blendshapes、landmarks、头部姿态进行平滑。
   - 提取 AU proxy、表情相关特征、时间变化特征。
   - 推荐使用低通滤波、One Euro Filter 或 exponential moving average。

4. **识别与映射层 Recognition / Mapping**
   - `blendshapes + landmarks -> AU proxy`
   - `AU proxy -> expression tendency`
   - `AU + expression + temporal dynamics + optional context -> appraisal proxy`

5. **状态层 State Store**
   - 高频状态：用于 3D avatar 或实时表情镜像，尽量绕开 React re-render。
   - 低频状态：用于 dashboard、规则判断、反馈策略。

6. **反馈层 Feedback**
   - 第一阶段：实时数值面板、AU 条形图、基础表情状态、Appraisal proxy 面板。
   - 第二阶段：2D/简化 avatar 或文字反馈。
   - 第三阶段：Three.js/R3F 3D 虚拟人，支持表情重定向和情境反馈。

7. **可选数据层 Optional Data Export**
   - CSV/JSON 导出作为可选研究功能。
   - 包含 session metadata、timestamps、AU proxy、expression scores、appraisal proxy、event markers。

## 4. 推荐 MVP 范围

### MVP 目标

优先证明这条链路可跑通：

**MediaPipe FaceLandmarker -> Blendshapes/Landmarks -> AU proxy -> 基础表情倾向 -> Appraisal proxy -> 简单反馈**

### MVP 必做功能

1. 摄像头初始化与权限处理。
2. FaceLandmarker 实时推理。
3. 提取并显示 52 个 blendshapes。
4. 实现一组核心 AU proxy：
   - AU1 inner brow raiser
   - AU2 outer brow raiser
   - AU4 brow lowerer
   - AU5 upper lid raiser
   - AU6 cheek raiser
   - AU7 lid tightener
   - AU9 nose wrinkler
   - AU12 lip corner puller
   - AU15 lip corner depressor
   - AU17 chin raiser
   - AU20 lip stretcher
   - AU23/24 lip tightener/pressor proxy
   - AU26 jaw drop
5. 基于 AU proxy 识别基础表情倾向。
6. 实现少量稳定 appraisal proxy：
   - pleasantness：主要由 AU6/AU12 与负性 AU 反向共同估计。
   - novelty/surprise：主要由 AU1/AU2/AU5/AU26 的突发上升估计。
   - obstruction/frustration proxy：主要由 AU4/AU7/AU23 与持续性负性模式估计。
   - low control / uncertainty proxy：主要由 AU20/AU26、头部姿态变化、表情不稳定度估计。
7. Dashboard：实时显示 AU、表情、appraisal proxy。
8. 简单反馈策略：当某一 proxy 持续超过阈值时触发提示或 avatar 状态变化。

### MVP 可选功能

- CSV/JSON 导出。
- event marker。
- IndexedDB 本地缓存。
- Pose/Hand。
- 3D avatar。
- 语音反馈。

## 5. 分阶段实施计划

## 第一阶段：Face Perception 与基础数据流

### 目标

跑通浏览器端人脸感知数据流，并建立稳定、可调试的实时特征输出。

### 技术选择

- Vite + React + TypeScript。
- `@mediapipe/tasks-vision`。
- Zustand 或轻量 event emitter 作为状态层。

### 核心任务

- 初始化摄像头。
- 加载 FaceLandmarker task model。
- 开启 `outputFaceBlendshapes: true`。
- 渲染 debug panel：视频、landmarks overlay、blendshape values。
- 建立 frame loop：控制推理频率，避免 UI 卡顿。
- 添加基础平滑器。

### 验收标准

- 主流桌面浏览器中可以稳定运行。
- 25–30 FPS 以上的 face tracking。
- blendshape 数据连续、无明显抖动。
- 关闭摄像头、切换页面、权限失败时有明确 fallback。

## 第二阶段：AU Proxy 层

### 目标

将 MediaPipe blendshapes 和 landmarks 映射为 FACS AU proxy。

### 原则

- 不声称完全等同于专业 FACS 编码。
- 每个 AU 输出 `0–1` 强度与 confidence。
- 每个 AU 的规则需要可解释、可调参。

### 示例映射

| AU | 名称 | 推荐输入特征 | 说明 |
|---|---|---|---|
| AU1 | Inner Brow Raiser | browInnerUp | 可直接来自 blendshape |
| AU2 | Outer Brow Raiser | browOuterUpLeft/Right | 可左右分开计算 |
| AU4 | Brow Lowerer | browDownLeft/Right | 与 AU1/AU2 有互斥关系 |
| AU5 | Upper Lid Raiser | eyeWideLeft/Right | 与 surprise/fear 相关 |
| AU6 | Cheek Raiser | cheekSquintLeft/Right | 与真实笑容相关 |
| AU12 | Lip Corner Puller | mouthSmileLeft/Right | 与 happiness 相关 |
| AU15 | Lip Corner Depressor | mouthFrownLeft/Right | 与 sadness 相关 |
| AU20 | Lip Stretcher | mouthStretchLeft/Right | 与 fear/uncertainty 相关 |
| AU23/24 | Lip Tightener/Pressor | mouthPressLeft/Right, mouthClose | MediaPipe 不一定直接区分，先做 proxy |
| AU26 | Jaw Drop | jawOpen | 与 surprise/fear 相关 |

### 验收标准

- AU 数值随用户动作变化方向正确。
- 左右侧 AU 能区分或合并。
- Dashboard 可以显示 AU proxy 强度。
- 规则参数集中配置，方便后续校准。

## 第三阶段：基础表情识别

### 目标

基于 AU proxy 输出基础表情倾向，而不是单一硬分类。

### 推荐输出

```ts
type ExpressionScores = {
  happiness: number;
  sadness: number;
  surprise: number;
  anger: number;
  fear: number;
  disgust: number;
  contempt: number;
  neutral: number;
};
```

### 原则

- 使用连续分数，不强制互斥。
- 输出 top emotion 仅用于 UI 显示。
- 保留 confidence 与 ambiguity 指标。

### 示例规则

- happiness：AU6 + AU12。
- sadness：AU1 + AU4 + AU15。
- surprise：AU1 + AU2 + AU5 + AU26。
- anger：AU4 + AU5/AU7 + AU23/24。
- fear：AU1 + AU2 + AU4 + AU5 + AU20 + AU26。
- disgust：AU9 + AU15/upper lip proxy。
- contempt：单侧 AU12/AU14 proxy。

### 验收标准

- 用户做典型表情时，目标表情分数明显上升。
- neutral 状态下各负性/正性分数不过高。
- 时间平滑后无频繁跳变。

## 第四阶段：Appraisal Proxy 层

### 目标

将 AU、表情倾向、时间变化与上下文事件结合，估计 appraisal proxy。

### 推荐维度

| 维度 | 是否 MVP | 输入依据 | 风险 |
|---|---|---|---|
| pleasantness | 是 | AU6/AU12、happiness、负性 AU 反向 | 相对可行 |
| novelty/surprise | 是 | surprise AU 组合的突发变化 | 相对可行 |
| obstruction/frustration | 是 | AU4/AU7/AU23 持续上升、negative expression | 中等风险 |
| low control / uncertainty | 是 | AU20/AU26、fear/sadness、头部/表情不稳定 | 中高风险 |
| goal attainment | 后续 | 需要任务上下文/用户行为结果 | 仅靠面部不足 |
| norm compatibility | 后续 | 需要社会/任务语境 | 仅靠面部不足 |

### 关键修正

- Appraisal 不应只由 AU 直接推出。
- MVP 中只实现 **appraisal proxy**，并明确来源。
- Goal attainment 与 norm compatibility 必须引入任务上下文、事件标记或用户自评，否则不应作为强结论输出。

### 推荐输出

```ts
type AppraisalProxy = {
  pleasantness: number;
  novelty: number;
  obstruction: number;
  lowControl: number;
  confidence: number;
  evidence: string[];
};
```

### 验收标准

- 每个 appraisal proxy 都能解释由哪些 AU/表情/时间特征触发。
- UI 中明确显示这是 proxy，而不是心理真值。
- 支持阈值与权重配置。

## 第五阶段：反馈策略

### 目标

根据识别结果进行实时反馈，使系统从“检测面板”变成“交互系统”。

### MVP 反馈

- 文本提示。
- 简单 2D 状态卡片。
- Dashboard 高亮。
- 可选音频/语音提示。

### 后续反馈

- 3D avatar 表情镜像。
- 低掌控/挫败 proxy 持续升高时，avatar 低头、语气柔和、给出帮助提示。
- pleasantness 高时，avatar 微笑或正向回应。
- novelty/surprise 高时，avatar 表现关注或询问。

### 反馈原则

- 不给出诊断式判断。
- 避免“你很焦虑/你很沮丧”这类强断言。
- 使用温和、可撤回的语言：
  - “看起来这个环节可能有些卡住，要不要我换一种提示？”
  - “我检测到一些 surprise/uncertainty 的信号，是否需要解释当前步骤？”

## 第六阶段：可选数据导出模块

### 定位

CSV/JSON 导出是研究增强功能，不是核心 MVP 阻塞项。

### 可选功能

- 开始/停止记录。
- 本地 session buffer。
- event marker。
- 导出 CSV。
- 导出 JSON。
- 可选 IndexedDB 本地缓存。

### 推荐数据结构

```json
{
  "sessionId": "...",
  "timestamp": 1234567890,
  "frameIndex": 1024,
  "blendshapes": { "jawOpen": 0.2 },
  "auProxy": { "AU12": 0.7 },
  "expressionScores": { "happiness": 0.8 },
  "appraisalProxy": { "pleasantness": 0.75 },
  "eventMarkers": []
}
```

### 隐私原则

- 默认不保存原始视频。
- 默认只保存数值特征。
- 导出前提示用户数据内容。
- 如果用于研究，需要 consent 文案与匿名化策略。

## 7. 技术风险与控制

## 7.1 Appraisal 推断过强

### 风险

仅凭面部表情无法可靠判断目标达成、规范违背、掌控感等复杂 appraisal。

### 控制

- 使用 proxy 表述。
- 加入 confidence 与 evidence。
- 对高风险维度延后实现。
- 后续引入任务上下文、用户自评或 event marker。

## 7.2 实时性能压力

### 风险

MediaPipe 推理、Dashboard 更新、3D 渲染同时运行可能导致卡顿。

### 控制

- MVP 不接入 3D avatar。
- Face 推理与 UI 更新分频。
- Dashboard 10Hz 左右即可。
- 3D avatar 后续使用 R3F `useFrame` 直接读高频 store。
- Worker 作为后续优化，不作为 MVP 前提。

## 7.3 AU 映射准确性

### 风险

MediaPipe blendshapes 不是完整 FACS 编码器。

### 控制

- 称为 AU proxy。
- 每个 AU 映射规则透明化。
- 支持校准页面。
- 保留 raw blendshapes，便于后续调整。

## 7.4 Avatar 模型资产不匹配

### 风险

3D 模型 morph targets 可能不符合 ARKit 52 blendshapes。

### 控制

- 3D avatar 延后到核心识别稳定后。
- 优先选择 ARKit-compatible GLB 或明确支持表情绑定的 VRM。
- 做单独 avatar retargeting adapter。

## 8. 建议里程碑

| 里程碑 | 内容 | 产出 |
|---|---|---|
| M1 | FaceLandmarker + camera loop | 可实时读取 blendshapes |
| M2 | AU proxy | AU dashboard 与配置文件 |
| M3 | 基础表情识别 | expression scores 与 top expression |
| M4 | Appraisal proxy | pleasantness/novelty/obstruction/lowControl |
| M5 | MVP feedback | 文本/面板反馈策略 |

| M7 | Avatar | 3D 表情镜像与状态反馈 |
| M8 | Pose/Hand | 多模态扩展 |

## 9. 总结

修订后的项目核心不是“直接从表情测量完整认知评估”，而是：

**以 MediaPipe 开源模型为感知基础，建立 AU proxy、基础表情识别、Appraisal proxy 估计与反馈交互的可解释 Web 系统。**

该定位更可实现，也更适合逐步研究验证。建议优先完成 Face -> AU -> Expression -> Appraisal proxy -> Feedback 的闭环；CSV/JSON 导出、3D avatar、pose/hand 作为后续增强模块逐步加入。

整体技术框架

Pipeline

1. 感知层

浏览器摄像头 + MediaPipe Web Tasks，本地模型资产运行，无需云端视频上传。

Camera→FaceLandmarkerPoseLandmarkerHandLandmarker→blendshapes / landmarks / worldLandmarks

2. 信号整理层

把 MediaPipe 原始输出转成 NAIM 内部统一帧：FaceFrame + PoseHandDebug。

raw blendshapes→AU proxy→Neutral Calibration→Per-AU Sensitivity

3. 认知/反馈层

当前仍是规则基线，但已经预留数据集驱动模型替换接口。

Expression Adapter→Appraisal Adapter→Feedback Policy→Dashboard / Gentle Prompt

4. Avatar 层

同时支持“意图驱动”和“直接信号驱动”。当前 VRM 动作用直接链路更敏捷。

blendshape → VRM expression+headPose+Kalidokit Pose/Hand→VRM bones

关键连接知识

MediaPipe / Kalidokit / VRM
MediaPipe

它在系统里负责“看见”

FaceLandmarker 输出 face blendshapes 和 facial transformation matrix。前者用于 AU/表情和 VRM 表情；后者用于更稳定的头部 yaw/pitch/roll。

PoseLandmarker / HandLandmarker 输出 2D landmarks 与 worldLandmarks。worldLandmarks 对姿态解算更重要，因为它保留了更接近 3D 的空间关系。

Camera → detectForVideo(video, now)
→ blendshapes / landmarks / worldLandmarks
→ NAIM FaceFrame
Kalidokit

它在系统里负责“解骨架”

Pose.solve 把 MediaPipe pose landmarks 转成 Spine、UpperArm、LowerArm 等旋转量。它不是 VRM 引擎，而是 retargeting 数学层。

Hand.solve 可解出手腕和手指，但默认 finger offset 对 AvatarSample_C 会让开放手指显得上翘，所以当前只保留 wrist，手指改为自定义 curl。

Pose.solve(world, image, { runtime: 'mediapipe' })
Hand.solve(handLandmarks, side)
→ rigRotation(vrmBone, solvedRotation)
VRM

它在系统里负责“显形”

@pixiv/three-vrm 提供 humanoid normalized bones 与 expression manager。NAIM 当前同时驱动 preset expressions 与 humanoid bones。

注意：不同 VRM 模型的骨骼轴向和默认姿态可能不同，所以“数学正确”不等于“视觉自然”。这就是 Finger axis test panel 的意义。

blendshape → expressionManager.setValue()
pose/hand → humanoid.getNormalizedBoneNode()
bone.quaternion.slerp(target)

下一步路线

Roadmap
优先

1. 校准手指轴向

使用 Finger axis test panel 找出 AvatarSample_C 上每根手指最自然的 axis/sign/gain，然后把最佳参数烘焙进自动 mapper。

优先

2. 真人实时手势验收

验证 open / fist / victory：V 手势应 index + middle 打开，ring + little 收拢；开放手应回到 exact rest。

调优

3. 镜像与左右手确认

当前按 Kalidokit selfie 示例交换左右手；若实测左右反了，只需翻转 handedness swap 策略。

后续

4. 数据集驱动 AU→Expression

目前规则层不再继续手调，等有标注数据后接入 ExpressionModelAdapter 替换 rule-based baseline。

后续

5. Appraisal proxy 优化

保留“proxy estimate”边界，不做心理真值宣称；未来可用多模态数据提升 pleasantness/novelty/control 等估计。

工程

6. Demo 汇报版本

把当前 NAIM 页面整理成稳定演示：启动提示、校准引导、信号状态灯、avatar 骨骼诊断和故障提示。

关键源码位置

  • src/hooks/useFaceTracking.ts：统一运行 Face / Pose / Hand landmarkers。
  • src/core/avatar/directBlendshapeToVrm.ts：MediaPipe blendshape → VRM preset expression。
  • src/core/avatar/directPoseHandToVrm.ts：Kalidokit pose/hand + 自定义 finger curl → VRM bones。

  • src/components/AvatarScene/FingerTestPanel.tsx：手指轴向手动校准面板。

已知边界

  • Appraisal 是 proxy estimate，不是心理诊断或心理真值。
  • Kalidokit 输出需要按具体 VRM 模型调轴向；AvatarSample_C 已做手臂竖直反转。
  • 手掌/手背判断来自 world-landmark palm normal proxy，仍需实测确认符号。
  • VRM 本地模型位于 public/models-local/，应保持 ignored，不提交。
